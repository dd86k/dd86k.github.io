Why a higher language? - 16/08/2015

It's very likely that you have heard about .NET, Mono, or even the Java Platform, which all have their own computing platform. Basically, instead of generating machine code for a specific operating system, the compiler generates intermediate code for its interpreter (often referred as a Virtual Machine).

So, why do some developers prefer those instead of lower languages?

  • One function, all platforms. (Availability)

    • If you write Console.WriteLine("Hi world!"); (C#), it will work on where ever the .NET framework or the Mono runtime can be installed on (also if you have access to a terminal or cmd). You don't need another compiler, then fill your code with preprocessor functions (#if) to establish OS specific code. Usually those language are meant to be multi-platform. Just look at Java: Phones, Smartphones, Windows, GNU/Linux, Mac OS X, Solaris, and the list goes on.

  • Oops! (Debugging)

    • If something goes wrong in your code, you can get the stack-trace really easily. Heck, the debugger shows what went wrong on your face. Debugging with an ease with IDEs, you get what went wrong on the spot. No longer have to deal with pointers (you can still use them in C#, both in safe (ref, IntPtr) and unsafe (unsafe {}, *) contexts) or segmentation faults (unspecific errors).

  • That was easy! (Flexibility)

    • C and C++ are considered a lot harder to learn. Robert Griesemer, at the GopherCon 2015 said that "It was clear what was wrong with C++ - it was very complex, hard to understand, missing concurrency support, didn’t scale, and had really long build times.". If you want, Go's history (Wikipedia) is a fun little read.

Although of having quite a few neat tricks, there are some downfalls here and there.

  • Hello World? (Frameworks)

    • "You need to install a framework!" -- Yes, you need the platform in order to compile and execute your code. Installing the a runtime (e.g. Java) may increase potential security risks (mostly looking at you, Flash). It is true, though the same can be achieved in just any language. Also while most would say that C#, or say, Java, isn't suitable for embedded devices -- I usually say this: There are smaller versions of the runtime specifically for embedded devices. I remember playing a Java Uno game on a flip phone, quite a few years ago.

  • zZz.. (Slower execution time)

    • "Waste of CPU cycles!" -- Yes, they do run a bit slower than C and C++. It's not a big difference too. Fortunately, managed languages do not need manual memory management. (Most of the time!)

  • Big, for you. (Memory consumption)

    • "Bloated!" -- Yes, they do consume more memory. Although, times have changed, most computers come with at least 2 GB of RAM, most Smartphones with 512 MB at a minimum.

Any "in between"?

There are some up-to-date languages that also compiles into machine code.

  • D

    • First appeared in 2001, D is like C#'s younger cousin. The syntax is almost the same. It's also available for .NET (compiles into IL instead of machine code). Personally (as myself, a C# dev), if I really worried about performance, I'd try out D.

    Hello World! example:

    import std.stdio;

int main(string[] argv)
{
    writeln("Hello World!");
    return 0;
}

    D is available on:

    • Ubuntu/Debian
    • Fedora/CentOS
    • openSUSE
    • FreeBSD
    • OS X
    • Windows

    Here is the Wikipedia page and their homepage.

  • Go

    • First appeared in 2009, Go is a language that was developed at Google. Syntax can look alike Javascript.

    Hello World! example:

    package main

import "fmt"

func main() {
	fmt.Println("Hello World!")
}

    Go is available on:

    • GNU/Linux
    • OS X
    • FreeBSD
    • OpenBSD
    • NetBSD
    • DragonFly BSD
    • Plan 9
    • Solaris
    • Windows

    Here is the Wikipedia page and their homepage.

There are a lot more out there of which escapes my mind.

C and C++ are still great languages. I never mentioned that you should avoid C, or C++. In the contrary, if you're willing to learn about C or C++, go ahead!

At the end of the day, it's all a matter of choosing your own language, and learning as much as you can. Learning a language and becoming the master of it. Don't be shy to explore other languages!

Programming languages are just like spoken languages, having more than one is always useful and fun.
