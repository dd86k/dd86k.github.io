Terminal tests - 06/08/2015

Quick post.

I did a few more "speed tests" between some terminals/command prompts.

The test, written in C# (no optimization), consists with rendering a bubble over a character 25000 times.

So it mostly tests the movement of the [terminal] cursor.

So here are the results! (Best to worst)

Terminals that fwod.exe could not run on and why.

  • (Linux/Mono) rxvt - System.ArgumentOutOfRange (Display buffer too small)
  • (Linux/Mono) Yakuake - System.ArgumentOutOfRange (Display buffer too small)
  • (Windows/.NET) ColorConsole - System.IO.IOException

If you want to test it yourself, you can download the exectutable with:

wget http://didi.wilomgfx.net/fwod.exe

Then run the test with the mono-runtime with:

~$ mono fwod.exe --speedtalktest

Or on Windows:

C:\> fwod.exe --speedtalktest
